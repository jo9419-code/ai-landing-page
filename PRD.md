# PRD: 개인 포트폴리오 랜딩페이지 (프리랜서 / AI 강사)

## 1. 개요
- **목적**: 프리랜서 AI 강사로서의 전문성을 소개하고, 방문자로부터 강의/컨설팅 문의를 유도하는 단일 페이지(정적 사이트)
- **형태**: 정적 HTML 1개 파일 (`index.html`), 별도 서버/백엔드 없음
- **언어**: 한국어
- **핵심 CTA**: "문의하기" 버튼 → `mailto:` 링크로 이메일 앱 연결

## 2. 디자인 톤앤매너
- **스타일**: 화려하고 생동감 있는 느낌 (그라디언트, 애니메이션, 컬러풀한 포인트 컬러 활용)
- **컬러/폰트**: 특정 지정 없음 — 제작자가 자유롭게 제안
  - 제안 방향: 밝고 에너지 있는 그라디언트 배경(예: 보라-핑크-오렌지 계열), 눈에 띄는 액센트 컬러, 가독성 좋은 산세리프 웹폰트
- **인터랙션**: 스크롤 시 요소 등장 애니메이션, 버튼 호버 효과 등으로 생동감 강조
- **반응형**: 모바일/태블릿/데스크톱 대응 필수

## 3. 페이지 구성 (섹션)

### 3.1 히어로 섹션
- 이름/브랜드명 (플레이스홀더, 추후 실제 정보로 교체 필요)
- 한 줄 소개 문구 (예: "AI를 쉽고 재미있게 가르칩니다")
- 부제/설명 텍스트
- 주요 CTA 버튼 ("문의하기" → mailto 링크)

### 3.2 소개/기능(서비스) 섹션
- 제공하는 강의/서비스 소개 (예: AI 활용 강의, 기업 교육, 1:1 컨설팅 등 — 실제 서비스 항목은 추후 확정 필요)
- 각 서비스별 간단한 설명 카드 형태

### 3.3 고객 후기 / 신뢰 지표 섹션
- 수강생/클라이언트 후기 카드 (이름, 소속, 후기 텍스트)
- 신뢰 지표(예: 누적 수강생 수, 강의 횟수, 협업 기업 로고 등) — 실제 수치/후기는 추후 제공 필요, 우선 플레이스홀더로 구성

### 3.4 FAQ 섹션
- 자주 묻는 질문 3~5개 (아코디언 형태로 클릭 시 펼쳐지는 UI)
- 예상 질문 예시: 강의 방식, 대상, 소요 시간, 비용 문의 방법 등 (실제 Q&A 내용은 추후 확정 필요)

### 3.5 문의(CTA) 섹션
- "문의/상담 신청" 강조 섹션
- mailto 링크 버튼 (이메일 주소는 플레이스홀더 → 추후 실제 이메일로 교체)

### 3.6 푸터
- 이메일, SNS/외부 링크(있는 경우) 텍스트/아이콘 노출
- 저작권 표기

## 4. 기술 요구사항
- 순수 HTML/CSS/JS 단일 파일 (`index.html`) — 외부 빌드 도구 불필요
- 외부 리소스는 최소화하되, 웹폰트/아이콘 등은 CDN 링크 또는 인라인으로 처리 가능
- 이미지가 필요한 경우 우선 플레이스홀더(또는 CSS로 대체 가능한 그래픽) 사용, 추후 실제 이미지로 교체
- 접근성: 시맨틱 태그, 적절한 alt 텍스트, 키보드 포커스 스타일 고려
- 문의 폼은 서버 없이 `mailto:` 링크로만 처리 (별도 폼 제출 로직 없음)

## 5. 추후 확정 필요 사항 (플레이스홀더로 우선 진행)
- 실제 이름/브랜드명
- 실제 이메일 주소
- 구체적인 강의/서비스 항목 및 설명 문구
- 실제 고객 후기 및 신뢰 지표 수치
- FAQ 실제 질문/답변 내용
- 프로필 사진 등 실제 이미지 에셋
- SNS/외부 링크 (블로그, 유튜브 등, 있는 경우)

## 6. 범위 외 (Out of Scope)
- 백엔드 서버, 데이터베이스, 폼 제출 API
- 다국어(영어 등) 버전
- 결제/구독 기능
- CMS 연동
